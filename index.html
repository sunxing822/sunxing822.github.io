Spring Boot Cache

Spring定义了CacheManager和Cache接口来统一不同的缓存技术。其中CacheManager是spring提供的各种缓存技术抽象接口,Cache接口包含了缓存的各种操作(增加,删除,获取缓存),一般情况下不会直接和这个接口打交道.

Srping支持的CacheManager的列表如下：
|CacheManager|描述|
|:–:|:—:|
|SimpleCacheManager|使用简单的Collection来存储数据,主要用于测试用途|
|ConcurrentMapCacheManager|使用ConcurrentMap来存储缓存数据|
|NoOpCacheManager|仅用于测试,不会实际存储数据|
|EhCacheCacheManager|使用EhCache作为缓存技术|
|GuavaCacheManager|使用Google Guava作为缓存技术|
|HazelcastCacheManager|使用Hazelcast作为缓存技术|
|JCacheCacheManager|支持JCache的标准的实现作为缓存技术|
|RedisCacheManager|使用Redis作为缓存技术|

声明式缓存注解

Spring提供了4个注解来声明缓存规则,声明式注解如下:

注解 解释
@Cacheable 在执行方法之前,Spring会先查看缓存中是否有数据,如果有数据,则直接返回缓存数据,若没有,则调用方法,将数据查询出来后放进缓存
@CachePut 无论如何都会将方法的返回值放入到缓存中
@CacheEvict 将一条或则多条数据从缓存汇总删除
@Caching 可以通过@Caching注解组合多个注解策略在一个方法上

声明式注解使用时配置内容为:

# 可选generic,ehcache,hazelcast,infinispan,jcache,redis,simple,guava,none
spring.cache.type= simple
# 程序启动时常见缓存的名称
spring.cache.cache-names=person
# ehcache 配置文件地址,其他文件的配置内容差不多
# spring.cache.ehcache.config=application.properties

声明式注解的属性值对应的意义：

属性 含义 默认值
propagationtion 定义了事务的声明周期,
REQUIRED: 方法A调用时没有事务,新建一个事务,如果方法A中调用了B方法,B方法沿用该事务,同时,如果B方法中发生异常需要数据回滚时,整个事务数据回滚。

REQUIRED_NEW:方法A,B不管有没有事务,都新建一个事务,这样B方法中发生异常需要回滚m不会影响到A方法

NESTED:和REQUIRED_NEW相同,但是支持JDBC,不支持jpa和hibernate

SUPPORTS:方法调用时,有事务就用,没有就不用。

NOT_SUPPORTED:强制方法不在事务中执行,若有事务,在方法调用到结束阶段,事务都将挂起。

NEVER:强制方法不在事务中执行,如果有则抛出一次。

MANDATORY: 强制方法在事务中执行,没有事务,则抛出一次		 REQUIRED
isolation Isolation(隔离)决定了事务的完整性,处理多事务对相同数据下的处理机制。

READ_UNCOMMITTED: 对于A事务里修改了一条记录,但是没有提交,B事务可以读取到修改以后的值。可导致脏读,不可重复读,以及幻读.

READ_COMMITTED: 在A事务中修改一条记录提交以后,B事务才可以读取到修改后的记录。阻止了脏读,但可能导致不可重复读和幻读。

REPEATABLE_READ: 不仅能实现READ_COMMITTED的功能,还能阻止A事务读取了一条记录,B事务将不允许修改这条记录,阻止了脏读和不可重复读,但是可能出现幻读.

DEFAULT: 当前数据库默认的数据隔离界.

SRIALZABLE: 此级别下,事务是顺序执行的,可以阻止上面的缺陷,但是开销非常大		 DEFAULT
timeout 事务过期时间,默认为当前数据库的事务过期时间
readOnly 当前事务是否是只读事务 false
rollbackFor 指定哪些异常是可以引起事务回滚的 Throwable
noRollbackFor 指定哪些异常是不会引起事务回滚的

自动配置ViewResolver

在控制器中一个方法返回一个字符串(视图名称)会根据BeanNameViewResolver去查找Bean的名称为返回字符串的View来渲染视图。如：

@Bean
public BeanNameViewResolver beanNameViewResolver(){
    BeanNameViewResolver resolver = new BeanNameViewResolver();
    return resolver;
}

定义一个View的Beanm名称为jsonView:

@Bean
public MappingJackson2JsonView jsonView(){
    MappingJackson2JsonView jsonView = new MappingJackson2JsonView();
    jsonView.setExtractValueFromSingleKeyModel(true);
    jsonView.addStaticAttribute("msg","hello world");
    return jsonView;
}

这个时候,如果我在对应的请求中,返回字符串:jsonView,就会使用上面定义的这个MappingJackson2JsonView这个对象,来渲染视图,说白了就是使用这个对象来处理视图,将其中的内容填充到视图对应的网页中.这个我们来看下具体的请求接口:

websocket为浏览器和服务端提供了双工异步通信的功能,浏览器可以向服务端发送消息,服务端也可以向浏览器发送消息.websocket是通过soket来实现双工异步通信能力的,但是直接使用WebSocket协议来开发程序比较繁琐,这里使用它的自协议STOMP.

广播式

广播式也就是服务端有消息时,会将消息发送给所有的链接了当前endpoint的浏览器.具体的配置很简单,需要先引入相应的依赖包:

<dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-websocket</artifactId>
</dependency>

在编写的过程中有两种方式可以选择:

1 使用SimpMessagingTemplate来发送消息
2 使用@MessageMapping和@SendTo注解来标记

两种方式中使用第二种更加简单,具体的@MessageMapping的用法和@RequestMapping相同。

1 当浏览器想服务端发送消息时,会映射到@MessageMapping对应的value地址中
2 当服务端想浏览器发送消息时,会映射到@SendTo对应地址中,这个地址在js中会有引用。

JavaNIO学习笔记(一) Java NIO简单概述

发表于 2016-07-18   |   分类于 java   |  

Java NIO(New IO)是一个可以替代标准Java IO API的IO API（从Java 1.4开始)，Java NIO提供了与标准IO不同的IO工作方式。

Java NIO由一下几个核心的组成部分：

Channels
Buffers
Selectors

1 Channels

Channels:管道,就是一个联通的通道,类似于流,但是又不相同

* 既可以从通道中读取数据,又可以写数据到通道中,但是流的读写是单向的
* 通道可以异步读写
* 通道中的数据总是需要先读到一个Buffer,或者是从一个Buffer中写入

JavaNIO下的最重要的通道的实现,基本的代码如下:

FileChannel                从文件中读写数据
DatagramChannel            能通过udp读写网络中的数据
SocketChannel            能通过TCP读写网络中的数据
ServerSocketChannel        可以监听新进来的tcp链接,想web服务器那样,对每一个新进来的链接都会创建一个ScoketChannel

2 Buffer

JavaNIO中缓冲区Buffer的主要功能用于和NIO通道进行交互,缓冲区本质上是一块可以写入数据,然后可以从其中读取数据的内存,这块内存被封装为NIO Buffer对象,并提供了一系列的方式,用于访问这块内存.

3 Selectors

Java NIO引入了选择器的概念，选择器用于监听多个通道的事件（比如：连接打开，数据到达）。因此，单个的线程可以监听多个数据通道。

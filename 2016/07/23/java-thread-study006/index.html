阻塞队列与普通队列的区别在于，当队列是空的时，从队列中获取元素的操作将会被阻塞，或者当队列是满时，往队列里添加元素的操作会被阻塞。试图从空的阻塞队列中获取元素的线程将会被阻塞，直到其他的线程往空的队列插入新的元素。同样，试图往已满的阻塞队列中添加新元素的线程同样也会被阻塞，直到其他的线程使队列重新变得空闲起来，如从队列中移除一个或者多个元素，或者完全清空队列

3 ArrayBlockingQueue

ArrayBlockingQueue 是一个有界的阻塞队列，其内部实现是将对象放到一个数组里.和缓冲区的构造思想相同,通过构建一个数组,在提供一些指针来表示不同操作,同样的和LinkedBlockingQueue一样,ArrayBlockingQueue也是线程安全的队列,在对其中数据进行操作的时候是进行加锁的.

来看下他的内部是怎么实现初始化的：

也就是在ArrayBlockingQueue的初始化中开发者必须指定队列的大小,jdk中没有提供默认的大小.这个是和其他的有点区别的。同样的,因为是数组,在java中数组的大小一旦指定以后是不能改变的,也就是这个队列的大小是定死了。从第二个构造函数中可以看到,指定了锁是公平还是非公平锁.公平锁死指按照FIFO的方式来分配锁,非公平锁死是按照优先级或者是线程锁获取次数,或者是自定义方式来分配。

同样的插入数据到队列中也是同样的,在插入时进行加锁,检测队列容量,如果当前队列满了,则等待出队动作移除数据后唤醒,同样的,如果阻塞了多个线程,其中一个线程检测发现已经有数据出队了,同样的插入数据后,队列还是没满,继续进行唤醒,唤醒阻塞的入队操作。

4 PriorityBlockingQueue

优先级队列,主要用于处理有优先级需求的数据,根据自定义的比较方式进行队列数据的排序。内部实现四通过封装一个数组来实现的队列,同样的内部有一个比较器,这个比较器是用来进行优先级的判定的。不同于ArrayBlockingQueue,PriorityBlockingQueue有一个默认的数组大小,为11。

PriorityBlockingQueue 是一个无界的并发队列。它使用了和类 java.util.PriorityQueue 一样的排序规则。你无法向这个队列中插入 null 值。所有插入到 PriorityBlockingQueue 的元素必须实现
java.lang.Comparable 接口。因此该队列中元素的排序就取决于你自己的 Comparable 实现。注意 PriorityBlockingQueue 对于具有相等优先级(compare() == 0)的元素并不强制任何特定行为。同时注意，如果你从一个 PriorityBlockingQueue 获得一个 Iterator 的话，该 Iterator 并不能保证它对元素的遍历是以优先级为序的。

5 DelayQueue

6 SynchronousQueue

SynchronousQueue 是一个特殊的队列，它的内部同时只能够容纳单个元素。如果该队列已有一元素的话，试图向队列中插入一个新元素的线程将会阻塞，直到另一个线程将该元素从队列中抽走。同样，如果该队列为空，试图向队列中抽取一个元素的线程将会阻塞，直到另一个线程向队列中插入了一条新的元素。

Hive学习笔记(二) Hive的数据类型和操作

发表于 2016-05-27   |   分类于 hive   |  

1 整型

整型数据可以指定使用整型数据类型，INT。当数据范围超过INT的范围，需要使用BIGINT，如果数据范围比INT小，使用SMALLINT。 TINYINT比SMALLINT小。

类型 后缀 示例
TINYINT Y 10Y
SMALLINT S 10S
INT - 10
BIGINT L 10L

2 字符串

字符串的类型可以使用单引号或者是双引号来指定,字符串类型的数据有两种:VARCHAR和CHAR. HIVE遵守C类型的转义字符。

类型 长度
VARCHAR 1 to 65535
CHAR 256

3 时间戳

它支持传统的UNIX时间戳可选纳秒的精度。它支持的java.sql.Timestamp格式“YYYY-MM-DD HH:MM:SS.fffffffff”和格式“YYYY-MM-DD HH:MM:ss.ffffffffff”。

4 日期

DATE值在年/月/日的格式形式描述 NaN.

5 浮点数

可以使用DECIMAL(进度,保留小数位)

6 数据库操作

1 创建数据库

CREATE DATABASE|SCHEMA [IF NOT EXISTS] <database name>

2 删除数据库

DROP DATABASE StatementDROP (DATABASE|SCHEMA) [IF EXISTS] database_name [RESTRICT|CASCADE];

删除非空数据库时需要加上参数：cascade

7 表操作

1 创建表

CREATE [TEMPORARY] [EXTERNAL] TABLE [IF NOT EXISTS] [db_name.] table_name
[(col_name data_type [COMMENT col_comment], ...)]
[COMMENT table_comment]
[ROW FORMAT row_format]
[STORED AS file_format]

例如：创建一个用户表,有姓名和年龄两个字段,建表语句如下:

create table person(
    name string,
    age int
)
filelds terminated by '\t'
lines terminated by '\n'
stored in text file

这个建表语句主要是制定了字段键分隔符和列之间分隔符以及存储的类型.
hive中默认的字段间分隔符为\001,行之间分隔符为\n.

2 删除表

drop table if exits person;

3 表分区

分区在文件存储的方式上主要呈现的就是讲数据按照不同的目录结构进行存储,比如在hive中按照天来进行分区,表存储目录结构为：

/user/hive/warehouse/test.db/test/dt=2016-05-21

每天的数据会在test下新建一个目录进行存储。

4 修改分区

ALTER TABLE table_name ADD [IF NOT EXISTS] PARTITION partition_spec
[LOCATION 'location1'] partition_spec [LOCATION 'location2'] ...;
partition_spec:
: (p_column = p_col_value, p_column = p_col_value, ...)

5 重命名分区

ALTER TABLE table_name PARTITION partition_spec RENAME TO PARTITION partition_spec;

6 删除分区

ALTER TABLE table_name DROP [IF EXISTS] PARTITION partition_spec, PARTITION partition_spec,...;

7 添加和删除字段

ALTER TABLE table_name ADD|REPLACE COLUMNS (col_name data_type[COMMENT col_comment], ...)

@Configuration
@EnableCaching
public class RedisConfig extends CachingConfigurerSupport {
@Bean
public CacheManager cacheManager(RedisTemplate redisTemplate) {
RedisCacheManager rcm = new RedisCacheManager(redisTemplate);
rcm.setDefaultExpiration(60);//设置缓存过期时间,单位为:秒
return rcm;
}

@Bean
public RedisTemplate<String, String> redisTemplate(RedisConnectionFactory factory) {
    StringRedisTemplate template = new StringRedisTemplate(factory);
    Jackson2JsonRedisSerializer jackson2JsonRedisSerializer = new Jackson2JsonRedisSerializer(Object.class);
    ObjectMapper om = new ObjectMapper();
    om.setVisibility(PropertyAccessor.ALL, JsonAutoDetect.Visibility.ANY);
    om.enableDefaultTyping(ObjectMapper.DefaultTyping.NON_FINAL);
    jackson2JsonRedisSerializer.setObjectMapper(om);
    //设置对象的序列化方式,转出json后存储
    template.setValueSerializer(jackson2JsonRedisSerializer);
    template.afterPropertiesSet();
    return template;
}

}

Spring-boot-mongodb简单的列子

spring-boot整合mongodb,基本和整合数据库的模式是相同的,具体的操作逻辑很简单。spring-boot中整合了mongodb使得整个过程很简单,首先需要集成相应的依赖包。

<dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-mongodb</artifactId>
    </dependency>

开发需要的依赖包有了,剩下的需要配置相应的mongodb的相应配置内容,如:

spring.data.mongodb.database=test
spring.data.mongodb.host=localhost
spring.data.mongodb.port=27017

由于是本地的环境,所以没有配置登录用户,如果没有设置,则不需要填写spring.data.mongodb.username的配置。配置好以后,剩下的就是配置相应的controller来进行测试和操作:

@RestController
@SpringBootApplication
public class SpringBootMongoApplication {

@Resource
private MongoTemplate mongoTemplate;

public static void main(String[] args) {
    SpringApplication.run(SpringBootMongoApplication.class, args);
}

@RequestMapping(value = "/test", method = RequestMethod.GET)
public Person test(@RequestParam String name,@RequestParam int age){
    Person person = new Person();
    person.setAge(age);
    person.setName(name);
    //将当前对象插入到mongodb中
    mongoTemplate.insert(person);

    //从mongodb中查询出来
    Query query = new Query();
    query.getQueryObject().put("age",age);
    return mongoTemplate.findOne(query,Person.class);
}
}

上面中通过传递过来的数据构造一个person对象,将这个对象插入到mongodb中,再通过查询的方式将对象查询出来传递回前端。具体的调用结果内容为:

Spring Boot基础学习(五) jpa的配置

发表于 2017-10-15   |   分类于 spring boot   |  

JPA的全称叫做Java Persistence API，JPA是一个基于O/R映射的标准规范，在这个规范中，JPA只定义标准规则，不提供实现，使用者则需要按照规范中定义的方式来使用。目前JPA的主要实现有Hibernate、EclipseLink、OpenJPA等，事实上，由于Hibernate在数据访问解决技术领域的绝对霸主地位，JPA的标准基本是由Hibernate来主导的

jpa是一个基本的框架逻辑,再使用spring-boot集成jpa时默认的做了Hibernate的实现.

Spring-boot-jpa简单的列子

首先创建一张表,表中用户表,具体的建表语句如下:

CREATE TABLE `test` (
      `name` varchar(255) NOT NULL,
      `id` int(11) NOT NULL,
      PRIMARY KEY (`name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

由于jpa采用了默认的Hibernate实现,因此只需要引入相应的依赖就可以实现操作数据库,具体的pom为:

上面的配置需要添加到application.properties中,数据源引入以后,jpa由于是框架模式,因此定义了一些基础的泛型接口,我们可以根据自己的需求定制自定义的接口,不过需要继承jpa框架中的定义.具体的如下:

我创建一个用户类,对应数据库中刚才创建的表:

@Entity
@Table(name = "test")
public class User {
@Id
private int id;
private String name;

public int getId() {
    return id;
}

public void setId(int id) {
    this.id = id;
}

public String getName() {
    return name;
}

public void setName(String name) {
    this.name = name;
}
}

@Entity这个注解标记当前的类为一个实体对象,同时@Table这个字段标记的是当前实体对象与数据库中标对应的关系,当然如果没有使用@Table标记,会找到当前实体对象的类名转换后和数据库中的表对应。

对当前这张表的操作的接口,由于jpa基本定义好了crud的操作,而且也提供了Hibernate的实现,具体的定义了操作的逻辑,对应的接口如下:

@Repository
public interface UserRepository extends JpaRepository<User,Integer>{}

Spring Boot操作数据库

SpringBoot整合jdbc

spring整合jdbc,让开发更加的简单方便,使用起来也很方便,配置起来也比较简单,首先需要增加相应的依赖:

<dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-jdbc</artifactId>
    </dependency>
    <dependency>
        <groupId>mysql</groupId>
        <artifactId>mysql-connector-java</artifactId>
        <version>5.1.44</version>
    </dependency>

这两个依赖添加以后,还需要在增加一个依赖,因为spring-boot中只是做了一个数据库操作的基本框架,但是数据库的连接池的管理是没有的,需要在使用时选择,这里选择了阿里的druid来处理的。

<dependency>
        <groupId>com.alibaba</groupId>
        <artifactId>druid</artifactId>
        <version>1.0.19</version>
    </dependency>

依赖添加好以后,剩下的是需要增加配置内容,下面为简单的配置内容为:

spring.datasource.url=jdbc:mysql://localhost:3306/spider
spring.datasource.username=root
spring.datasource.password=root
spring.datasource.driver-class-name=com.mysql.jdbc.Driver

依赖添加好以后,剩下的就是在注册数据库连接池的bean,如下:

@Autowired
private Environment env;

@Bean
public DataSource dataSource() {
    DruidDataSource dataSource = new DruidDataSource();
    dataSource.setUrl(env.getProperty("spring.datasource.url"));
    dataSource.setUsername(env.getProperty("spring.datasource.username"));//用户名
    dataSource.setPassword(env.getProperty("spring.datasource.password"));//密码
    dataSource.setDriverClassName(env.getProperty("spring.datasource.driver-class-name"));
    dataSource.setInitialSize(2);
    dataSource.setMaxActive(20);
    dataSource.setMinIdle(0);
    dataSource.setMaxWait(60000);
    dataSource.setValidationQuery("SELECT 1");
    dataSource.setTestOnBorrow(false);
    dataSource.setTestWhileIdle(true);
    dataSource.setPoolPreparedStatements(false);
    return dataSource;
}

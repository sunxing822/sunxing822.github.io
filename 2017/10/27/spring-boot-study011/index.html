自动配置ViewResolver

在控制器中一个方法返回一个字符串(视图名称)会根据BeanNameViewResolver去查找Bean的名称为返回字符串的View来渲染视图。如：

@Bean
public BeanNameViewResolver beanNameViewResolver(){
    BeanNameViewResolver resolver = new BeanNameViewResolver();
    return resolver;
}

定义一个View的Beanm名称为jsonView:

@Bean
public MappingJackson2JsonView jsonView(){
    MappingJackson2JsonView jsonView = new MappingJackson2JsonView();
    jsonView.setExtractValueFromSingleKeyModel(true);
    jsonView.addStaticAttribute("msg","hello world");
    return jsonView;
}

这个时候,如果我在对应的请求中,返回字符串:jsonView,就会使用上面定义的这个MappingJackson2JsonView这个对象,来渲染视图,说白了就是使用这个对象来处理视图,将其中的内容填充到视图对应的网页中.这个我们来看下具体的请求接口:

@RequestMapping(
        value = "/test/json",
        method = RequestMethod.GET,
        produces = {MediaType.APPLICATION_JSON_VALUE}
)
public String testJosn(Model model){
    User user = new User();
    user.setName("sunx");

    model.addAttribute("user",user);

    return "jsonView";
}

自动配置静态资源

静态资源的配置,系统默认的配置路径地址为/static或是/public.具体的可以通过自动配置类中增加相应的配置,如:

@Configuration
public class DemoWebMvcAutoConfiguration extends WebMvcConfigurerAdapter {
    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/css/{filename:\\w+\\.css}");
        registry.addResourceHandler("/css/**");
    }
}

上面的列子中可以配置自定义的目录地址到系统中~或则是使用系统默认的路径地址~

自动配置Formatter

自定义的格式化工具,首先需要基层Formatter接口,具体的如下,下面为在字符串前面增加内容:

public class StringFormatter implements Formatter<String> {
        @Override
    public String parse(String s, Locale locale) throws ParseException {
        return "__" + s;
    }

    @Override
    public String print(String s, Locale locale) {
        return s;
    }
}

1 spring-boot消息转化器是组合操作,根据方法的参数类型来定,上面的代码中将集合中的消息转换器清空了(清空了集合)后添加了一个字符串的转化方式,调用测试接口传递一个对象过来时就无法转化抛异常
2 调用测试接口3时,使用的注解是@RequestParam来读取采纳数的,这个时候没有做消息转换器,使用@RequestBody来注解参数时,会走消息转换器转化,可见除了需要指定SupportedMediaType(在消息转换器中有getSupportedMediaTypes()这个方法)外,不同的注解消息转换器的方式不一样,消息转换器不支持@RequestParam.
3 @RequestParam只适用于处理基础类型的数据以及字符串类型的数据,同时常用与Get请求中的基础参数的获取或则是表单提交时的基础类型的参数取值。@RequestBody则用于请求体的处理,基本将整个请求的内容当成一个整体进行处理。

Tomcat相关配置

toomcat常用的配置内容:

server.port= # 服务监听的端口
server.session-timeout= # 用户的默认回话超时时间
server.context-path= # 配置访问路径,默认的为 /
server.tomcat.uri-encoding= # 配置tomcat的编码格式,默认为UTF-8
server.tomcat.compression= # tomcat是否开启压缩,默认为关闭状态

使用代码来控制相应的配置内容:

public class SpringBootWebApplication implements EmbeddedServletContainerCustomizer{
    @Override
    public void customize(ConfigurableEmbeddedServletContainer container) {
        //设置端口
        container.setPort(9090);
        //设置session的过期时间
        container.setSessionTimeout(1000 * 60 * 60);
    }
}

替换tomcat为jetty:

<dependency>
        <groupId>org.springframework.boot</groupId>

SSL配置:
使用java自带的工具生产相应的ssl证书,具体的过程为:

具体的参数意义为:

-alias test(别名) 
-keypass ***(别名密码) 
-keyalg RSA(算法) 
-keysize 1024(密钥长度) 
-validity 365(有效期，天单位) 
-keystore /data/study/key/test.keystore(指定生成证书的位置和证书名称) 
-storepass(获取keystore信息的密码)

使用openssl来生成认证证书

openssl genrsa -des3 -passout pass:x -out server.pass.key 2048
openssl rsa -passin pass:x -in server.pass.key -out server.key
rm server.pass.key
openssl req -new -key server.key -out server.csr
openssl x509 -req -days 365 -in server.csr -signkey server.key -out server.crt

这里采用openssl生成的证书,这个证书是中间证书,需要转化为java中可以识别的证书格式,具体转化过程为:

openssl pkcs12 -export -name server -in server.crt -inkey private.key -out keystore.p12
keytool -importkeystore -destkeystore keystore.jks -srckeystore keystore.p12 -srcstoretype pkcs12 -alias server

spring boot配置ssl:

# 配置ssl
server.port = 9443
server.ssl.enabled = true
server.ssl.key-store = classpath:key/keystore.jks
server.ssl.key-store-password = sunxing
server.ssl.key-store-type = JKS
